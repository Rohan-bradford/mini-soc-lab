# Playbook: Web Attack

1. Identify source, URI, method, response status, user agent, and request volume.
2. Determine whether the source is the approved lab tester.
3. Correlate reverse-proxy, application, Suricata, and host logs.
4. Check whether suspicious requests produced authentication, file, process, or database events.
5. Preserve relevant logs and container state.
6. Block the source or stop the disposable application if impact is observed.
7. Patch, reconfigure, or rebuild; then repeat the exact test safely.
8. Document exploitability, impact, root cause, and rescan evidence.
