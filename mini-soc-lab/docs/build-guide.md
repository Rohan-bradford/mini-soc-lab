# Live Lab Build Guide

## 1. Prepare the Host

Recommended capacity:

- 16 GB RAM minimum; 24-32 GB is more comfortable
- 100 GB free disk
- Hardware virtualization enabled
- VirtualBox, VMware, Hyper-V, or another trusted hypervisor
- Docker on the Ubuntu SIEM VM if using containerized Wazuh

## 2. Create the Isolated Network

Create `SOC-LAB` as host-only/internal networking with `10.10.10.0/24`. Disable promiscuous
mode except where the Suricata sensor requires traffic visibility. Add temporary NAT adapters
only for updates, then disconnect them.

Test that:

- Lab VMs can reach the SIEM.
- Vulnerable services are not reachable from another physical device.
- Dashboard administration is reachable only from the host/analyst path.

## 3. Deploy Wazuh

Choose one:

- Official all-in-one installation assistant
- Official Wazuh Docker single-node deployment

Use [deploy/wazuh/README.md](../deploy/wazuh/README.md). Record exact versions, memory,
deployment date, and credential changes in a private build log.

## 4. Enroll Endpoints

Install agents using the commands generated by the Wazuh dashboard. Use meaningful agent
names and confirm active status before adding custom telemetry.

## 5. Configure Windows

1. Apply updates and take a snapshot.
2. Enable advanced audit policy for logon and account management.
3. Enable PowerShell logging.
4. Install/configure Sysmon.
5. Merge the event-channel collection blocks.
6. Confirm Event IDs 1, 4625, and 4732 in Wazuh.

## 6. Configure Linux and Network Telemetry

1. Collect `/var/log/auth.log`.
2. Install Suricata on the sensor or Ubuntu endpoint.
3. Enable EVE JSON DNS, HTTP, TLS, flow, alert, and file records.
4. Test Suricata configuration before restarting.
5. Forward EVE JSON through the Wazuh agent.

## 7. Deploy Juice Shop

Use the pinned Compose file and bind only to loopback or the isolated VM address. Verify it is
not reachable from public/Wi-Fi interfaces.

## 8. Deploy Rules

Add one rule at a time:

1. Validate syntax.
2. Generate a positive event.
3. Generate a benign comparison event.
4. Record the alert and raw event.
5. Tune fields and thresholds.
6. Document false positives.

## 9. Build Dashboards

Use `dashboards/dashboard-spec.json` and `queries.md`. Export saved objects from the live
dashboard after removing credentials and environment-specific identifiers.

## 10. Capture Evidence

For each scenario capture:

- Raw event excerpt
- Alert
- Search query
- Timeline
- Dashboard panel
- Incident report
- Rule and tuning notes
- Rollback/remediation evidence

Place genuine screenshots under `dashboards/screenshots/live/` and clearly label dates and
versions.
