# Live Evidence Checklist

The current repository contains synthetic telemetry and a dashboard design mockup. Complete
these tasks on your hardware before describing the project as a deployed home lab.

- [ ] Record hypervisor and VM specifications.
- [ ] Capture the isolated virtual network configuration.
- [ ] Deploy Wazuh and record the exact version.
- [ ] Change all default credentials.
- [ ] Enroll Windows and Ubuntu agents.
- [ ] Verify Sysmon, Security, PowerShell, auth.log, and Suricata ingestion.
- [ ] Run all eight authorized simulations.
- [ ] Capture raw event and alert screenshots for each rule.
- [ ] Export the dashboard saved objects.
- [ ] Replace `soc-overview.svg` with a live screenshot, retaining the mockup as a design file.
- [ ] Update incident reports with actual timestamps and event IDs.
- [ ] Perform a scoped vulnerability scan.
- [ ] Remediate at least one finding and attach rescan evidence.
- [ ] Run the hardening checklists.
- [ ] Sanitize every artifact before committing.

Never fabricate live evidence. A transparent build-in-progress is stronger than an
unverifiable claim.
