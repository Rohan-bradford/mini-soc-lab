# SIEM Hardening Checklist

- [ ] Change all vendor/default credentials.
- [ ] Restrict dashboard/API access to management interfaces.
- [ ] Use TLS and protect private keys outside Git.
- [ ] Use named analyst accounts and least privilege.
- [ ] Synchronize time on every component.
- [ ] Set disk, shard, and retention alerts.
- [ ] Back up rules, decoders, dashboard exports, and configuration.
- [ ] Review agent enrollment and remove stale agents.
- [ ] Test alert routing without sending sensitive data.
- [ ] Record administrative changes.
- [ ] Keep the Wazuh stack and host patched.
- [ ] Test restoration from backup.
